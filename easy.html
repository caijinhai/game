git相关命令

这里是文章的smarry简介。获取文章的中心意思。抓住别人，促使去看文章 

1.安装：Windows下安装git：git->git bash ;
git config --global user.name "Your  Name" 
git config --global user.email "email@example.com"


2.创建版本库：创建一个新目录；在目录下git bash命令框git init 初始化（.git）;将文       
                     件放进repositories（仓库）git add "file"  git commit -m "注释"


3.修改文件：查看当前状态  git status       查看文件改动 git diff "file"
                  

4.版本回退：查看历史 git log     简化信息 --pretty=oneline   
 （当前版本HEAD   上一版本HEAD^   上上版本HEAD^^   往上100版本           HEAD~100） 
回退上一版本：git reset --hard HEAD^（也可写版本ID git reset --hard 3628164） 
 记录命名 git reflog


5.工作区和暂存区： working dirctory ->License->repository


6.管理修改：查看当前版本与仓库版本不同：git diff HEAD -- "文件"
不add到暂存区就不会commit到仓库中


7.撤销修改：修改后没放到暂存区  git checkout 
 修改后放到了暂存区  git reset HEAD file 把暂存区的修改撤销掉（unstage）
                                git checkout 


8.删除文件：删除版本库中文件git rm file       git commit -m "remove file "
误删后恢复（从版本库还原或替换）git checkout -- file

9.远程仓库： 一. 创建 ssh key   ：ssh-keygen -t  rsa -C“YourEmail”
二。登录github   Add ssh key 进行匹配

10.添加远程库:一。本地库推送  git remote add origin  htttps://github.com/your name/learngit.git
git push -u origin master（master 代表分支）
以后只要本地做了修改，只要git push origin master即可 ，远程同步
二。远程克隆：git clone htttps://github.com/your name/learngit.git


11.创建并合并分支：git checkout -b  分支（branch）（git branch dev  , git checkout dev ）
git branch  查看当前分支
git add file     git  commit -m ""  分支完成之后提交
切回到master分支再合并dev   git checkout  master      git merge dev  合并之后删除分支dev   git branch -d  dev  


12.解决冲突：手动解决 打开文件手动解决再提交再删除分支
查看分支合并情况 git log


13.分支管理策略：禁止快速合并情况（fast forward）git merge --no-ff -m "merge with no-ff" dev


14.bug分支：储存当前工作git stash
查询当前储存的工作 git stash  list
恢复工作区git stash apply  但是没有删除stash用git stash drop删除
恢复工作区 git stash pop 删除了trash
恢复指定stash   git stash apply stash@{  ID号}

15.Feature分支 : 当没有合并的分支要删除时就需要git branch -D dev


16.多人协作 ： 获取远程库信息： git remote或者git remote -v
创建远程origin的dev分支到本地 ： 克隆之后 git checkout -b dev origin/dev
最新提交和你试图推送提交有冲突：本地与远程建立连接 git branch --set-upstream dev origin/dev  ，  抓取新的提交 git pull ,合并冲突，提交，push


17.创建标签：git tag （git  tag  v1.0）
查看标签 git tag
找到历史提交id    git log --pretty=oneline --abbrev-commit
对应ID打标签 git tag v1.0 200245
查看标签信息 git show 
用私钥签名一个标签 git tag -s v0.2 -m "signed version 0.2 released" fec145a  签名采用PGP签名，因此，必须首先安装gpg（GnuPG），如果没有找到gpg，或者没有gpg密钥对，就会报错


18.操作标签： 删除标签 git tag -d v1.0
推送某个标签到远程，使用命令git push origin 
一次性推送全部尚未推送到远程的本地标签：git push origin --tags
删除远程标签：git tag -d 《atgname》再git push origin  :refs/tags/v0.9


19.自定义git：让git显示颜色git config --global color.ui true


20.配置别名：git config --global alias.st status  以后st就表示status
git config --global alias.lg "log

document.getElementById("save").onclick = function(){
     //发送ajax请求
     var request = new XMLHttpRequest();
     request.open("POST","service.php");
     var data = "name="+document.getElementById("staffName").value
                    +"&number="+document.getElementById("staffNumber").value
                    +"&sex="+document.getElementById("staffSex").value
                    +"&job="+document.getElementById("staffJob").value;    
     request.setRequestHeader("Content-Type","application/x-www-form-urlencoded");
     request.send(data);
     request.onreadystatechange  = function(){
          if(request.readyState===4){
               if(request.status===200){
                    document.getElementById("save-result").innerHTML=request.responseText;
               }else{
                    alert('发生错误：'+request.status);
               }
          }
     }

}
